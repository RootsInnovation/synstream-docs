# Deployment

Learn about deployment features in the Synstream workflow designer header.

## Overview

Deployment features allow you to:

- Deploy workflows to production
- Select deployment environments
- Monitor deployment status
- Manage deployment configurations

## Deploying Workflows

### Deployment Process

1. Click Deploy button in header
2. Select deployment environment
3. Review deployment configuration
4. Confirm deployment
5. Monitor deployment status

### Deployment Environments

Available deployment environments:

- **Development** — development environment
- **Staging** — staging environment
- **Production** — production environment

## Deployment Configuration

### Configuration Options

Deployment configuration includes:

- Environment selection
- Resource allocation
- Scaling settings
- Monitoring settings
- Notification settings

### Deployment Settings

Configure deployment settings:

- Deployment name
- Deployment description
- Resource limits
- Scaling rules
- Health checks

## Deployment Status

### Status Monitoring

Monitor deployment status:

- Deployment progress
- Deployment logs
- Error messages
- Success notifications

### Deployment History

View deployment history:

- Previous deployments
- Deployment dates
- Deployment status
- Rollback options

## Deployment Best Practices

### Pre-Deployment Checklist

Before deploying:

- Test workflow thoroughly
- Review deployment configuration
- Check resource availability
- Verify environment settings

### Post-Deployment Monitoring

After deploying:

- Monitor deployment status
- Check workflow execution
- Review logs and metrics
- Handle errors promptly
